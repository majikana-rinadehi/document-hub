## 概要


Claude Code から Cursor に移行する際、カスタムスラッシュコマンドを引き続き使いたいと思ったのが、この記事を書くきっかけでした。実際に試してみたところ、基本的な使い方は Claude Code とほぼ同じで、移行はスムーズにいきそうです。


### 基本的な仕様

- **2026/01/04時点ではまだベータ版**です
- 基本的には Claude Code 同様、`.cursor/commands` にマークダウンを作成していきます
- チャットで `/` 入力で使えます
- Claude Code のようなフロントマター構文はまだサポートしていません

実際に試してみた感じでは、フロントマターがない以外は同じように使えるので、移行コストは低そうです。フロントマターについては、公式ドキュメントを見る限り、近日中にサポートされそうな雰囲気ですね。


## コードレビューコマンドを使ってみる


実際にコードレビューコマンドを作成して使ってみました。GitHub のサンプルを参考にしながら、自分の環境で動作確認してみたので、その手順を共有します。


### 実装手順


[コードレビューコマンドのサンプル](https://github.com/hamzafer/cursor-commands/blob/main/.cursor/commands/code-review.md) をコピーして、ホームディレクトリに以下のファイルを作成しました：


```plain text
~/.cursor/commands/code-review.md
```


個人開発のリポジトリで適当なPRを作成して、チャットタブで `/code-review {PRリンク}` と入力してみました。

指定したPRのコードレビューがチャットに出力されました。実際に使ってみると、PRのリンクを指定するだけで、コードレビューが自動生成されるので便利ですね。Claude Code で使っていたコマンドとほぼ同じ感覚で使えるので、移行のストレスはほとんどありませんでした。


## まとめ


Cursor の Commands 機能を試してみた結果、Claude Code からの移行は思っていたよりも簡単でした。


### 主なポイント

- **Claude Code とほぼ同様に使える** - 既存のコマンドファイルがそのまま使える可能性が高い
- **フロントマターなどは近日サポートされそう** - 公式ドキュメントを見る限り、機能追加が予定されている

現時点ではベータ版ということもあり、フロントマター構文など一部の機能はまだサポートされていませんが、基本的な使い方は問題なく、移行を進められそうです。今後のアップデートに期待しつつ、現状の機能で十分に活用できそうです。
